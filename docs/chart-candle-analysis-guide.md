# 차트 캔들·신호 기능 설명서 (사용자용)

**앱에서 보기:** 메인 화면 → 차트 **라벨 설정** → 「차트 캔들·신호 설명서 전체」또는 주소 `/help/chart-candle-analysis` (내용 동기화 시 `app/help/chart-candle-analysis/page.tsx` 함께 수정)

이 문서는 **ailongshort 웹 차트**에 표시되는 캔들 분석·마커·보조 신호가 **코드상 어떤 의미인지** 정리한 것입니다.  
**투자 권유가 아니며**, 실제 매매는 본인 판단·책임 하에 하시고, 손절·포지션 크기 등 **리스크 관리**를 반드시 병행하세요.

---

## 1. 차트에 나오는 것 한눈에 보기

| 표시 | 대략적인 의미 | 비고 |
|------|----------------|------|
| **L** / **S** | 롱·숏 쪽 **메인 신호 마커** (확정 신호, RSI 다이버전스·상단 판정 등이 겹쳐 그려짐) | 최근 구간에서는 상단 최종 방향과 **반대 마커는 숨기는** 로직이 있음 |
| **L·숫자** / **S·숫자** | A(메타) ON일 때: 상단 **신뢰도(confidence)** 또는 RSI 쪽 **totalScore**가 붙은 경우 | 숫자 = 수익 보장 아님 |
| **🚀** / **📉** | **구조 로켓** — BOS·존 반응·CHOCH 등 구조 이벤트에 따른 강조 마커 | 로켓이 있으면 같은 봉에서 L/S 원과 **한 덩어리로** 정리되는 경우가 많음 |
| **C↑** / **C↓** / **c+** / **c-** | **캔들 점수 보조 마커** (아래 2절 참고) | **캔들신호** + 분석 설정이 켜져 있을 때 |
| **C↑78** 형태 | A(메타) ON일 때 점수 숫자 표시 | 78 = 그 봉의 **캔들 점수**(대략 0~100) |
| **T↑** / **T↓** | **타이롱 종가** 계열 엔진이 마지막 봉에서 본 방향 힌트 | 보조 신호 |
| **존·라벨·피보 등** | 공급/수요, FVG, OB, 반응구간 등 **오버레이** | 색·라벨은 설정·엔진 종류에 따름 |

**B(클릭)** 을 켠 상태에서 봉을 누르면, 그 시각에 붙은 마커를 **한글 한 줄 요약**으로 볼 수 있습니다.

## 2. 캔들 분석 점수 (차트의 C↑ / C↓ / 숫자)

엔진 파일 `engine/candles/candleEngine.ts`의 **`scoreCandles`** 로 각 봉마다 점수를 냅니다.

### 2.1 상승 봉 vs 하락 봉

- **상승(불) 캔들**: `종가 ≥ 시가` 인 봉을 **bullish** 로 봅니다.  
- **하락(베어) 캔들**: `종가 < 시가` 이면 **bearish** 입니다.  
- 마커 위치: 불일 때는 보통 **캔들 아래(belowBar)**, 베어일 때는 **위(aboveBar)** 에 보조 점이 붙습니다.

### 2.2 점수는 어떻게 쌓이나 (기준 50에서 출발)

한 봉에 대해 대략 다음이 **가감**됩니다 (모두 해당될 수 있음).

- **긴 실체** (전체 레인지 대비 몸통 비율이 크면) → 가점  
- **불일 때 종가가 위쪽 근처** / **베어일 때 종가가 아래쪽 근처** → 가점  
- **엔걸핑** (전 봉 몸통을 삼키는 형태) → 가점  
- **브레이크아웃** (전 고/저를 종가로 넘김) → 가점  
- **실패한 돌파** (꼬리만 넘기고 몸통은 못 넘김) → 감점  
- **긴 윗꼬리·아래꼬리(스윕 의심)** → 소량 가점  
- **망치 / 역망치 형태** → 소량 가점  

최종 점수는 **0~100** 으로 잘립니다.

### 2.3 강도 구간 (엔진 내부)

- **strong**: 점수 **≥ 75**  
- **normal**: **≥ 55**  
- **weak**: 그 미만  

### 2.4 차트에 C 마커가 뜨는 조건

`lib/candleLsMarkers.ts` 기준:

- **강한 쪽만** 후보: `strength === 'strong'` 이거나 점수 **≥ 62** 또는 **≤ 38** 인 봉  
- **C↑ / c+**: **불(bullish)** 쪽으로 분류된 봉 + 위 조건  
- **C↓ / c-**: **베어(bearish)** + 위 조건  
- **62 미만이지만 38 초과** 인 “중간” 봉은 마커로 안 나올 수 있습니다 (필터 때문).

즉, **C와 숫자는 “그 순간 캔들 형태·모멘텀 점수”**이지, 단독으로 “여기서 무조건 진입”을 뜻하지는 않습니다.

## 3. 지지·저항이 “좋다/나쁘다”를 앱이 어떻게 보는지

앱은 **가격대(존)·스윕·다이버전스** 등을 **점수/체크리스트**에 녹입니다.

- **수요 존(demand)·지지 근처** + 롱 판정이 맞을 때 → RSI 다이버전스 쪽 점수에 **존 가점**이 붙는 식의 로직이 있습니다 (`divergenceSignalEngine` 의 Zone 항목).  
- **공급 존(supply)·저항 근처** + 숏 판정이 맞을 때 → 마찬가지로 숏 쪽에 유리하게 작동할 수 있습니다.  
- **유동성 스윕**(매도/매수 스윕)이 잡히면 체크리스트에 반영됩니다.

**중요:** “좋은 지지”라고 그린 구간도 **언제든 이탈**할 수 있습니다. 오버레이는 **참고용 레이어**이고, **확정적 예측**이 아닙니다.

## 4. 어디서 롱·숏을 보나 (메인 L / S)

메인 **L·S** 는 대략 다음이 합쳐집니다.

1. **4요소 등으로 “확정”** 된 방향이 있으면 그 **신호 봉 시각**에 고정되는 맵  
2. **RSI 다이버전스 신호** (롱/숏/관찰)와 **다이버전스 라인 끝점**  
3. 상단 **최종 verdict** 가 롱/숏이면 **마지막 봉**에도 RSI 맵으로 보강  
4. 서버에 쌓인 **확정 히스토리**를 같은 심볼 차트에 **백필**  
5. **구조 로켓(🚀📉)** 이 같은 봉에 있으면 **로켓이 우선**되고, 선확과 방향이 어긋나면 **확정/RSI 쪽**을 택하는 등 **한 봉에 하나**로 정리하는 규칙  

최근 약 **22봉** 구간에서는 **상단 최종 방향과 반대**인 L/S·로켓·선확을 **숨겨** 혼선을 줄입니다.

**실제 진입가·손절**은 우측 **실행/브리핑·구조 세트업** 등 **별도 패널·라인**을 함께 보는 것이 안전합니다. 차트의 L/S 원만으로 “청산까지” 판단하기는 어렵습니다.

---

## 5. 타이롱 (T↑ / T↓)

엔진 **`tailongCloseSignals`** 가 **마지막 봉**에서만 방향 힌트를 주면 **T↑ / T↓** 가 붙을 수 있습니다.  
**캔들 점수(C)** 와는 **다른 파이프라인**이므로, 둘이 다를 수 있습니다.

---

## 6. 설정 A · B · C (라벨 설정 패널)

| 옵션 | 역할 |
|------|------|
| **A 메타** | L/S에 신뢰도·점수 접미사, C/T 마커에 숫자·텍스트 보강 |
| **B 클릭** | 봉 클릭 시 **이 봉에 붙은 마커 요약** 패널 |
| **C 밀도** | 켜면 **L/S · 로켓 · 보조(C·T) · 선확** 을 **개별 ON/OFF** (차트가 복잡할 때) |

**보조 마커 최대 개수** 슬라이더는 C/T 같은 보조 마커가 너무 많이 찍히지 않게 **상한**을 둡니다.

## 7. “그래야만 안전한가?”에 대한 짧은 답

**아니요.** 이 앱은 **분석·시각화 도구**이며:

- 과거·백테스트·시뮬 결과는 **미래 수익을 보장하지 않습니다**.  
- **확정 신호·높은 점수·로켓**이 있어도 **연속 손실·갭·유동성** 등으로 손실이 날 수 있습니다.  
- **안전에 가깝게** 가려면 보통  
  - **손절(또는 리스크 한도)을 먼저 정하고**,  
  - **상위 타임프레임 추세**와 **하위 진입 타이밍**을 나눠 보고,  
  - **한 지표에 올인하지 않고** 존·구조·캔들·신호를 **겹쳐서** 보며,  
  - **본인 규칙(포지션 크기·일일 손실 한도)** 을 지키는 방식이 권장됩니다.

앱의 **학습 필터·과거 SL 실패 숨김** 등은 “조금 덜 나쁜 조건”을 고르기 위한 **보조**일 뿐, **무위험**을 만들지는 못합니다.
